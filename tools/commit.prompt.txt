# TOOL: git commit

## INTENT
Draft a high-signal, BlockSmith-aligned git commit message for the specified changes.

## STRATEGY (The Research Phase)
1.  **Analyze Context**: Run `git status` and `git diff` on the target files/directory.
2.  **Review History**: Run `git log -n 3` to match the project's commit style (author, formatting, verbosity).
3.  **Synthesize Change**: Summarize the technical "What" and the philosophical/strategic "Why".

## FORMAT (The Voice)
- **Title**: `<type>: <Short descriptive title>` (Types: Feat, Fix, Docs, Refactor, Chore).
- **Body**: Bulleted list of technical changes.
- **Footer**: Philosophical anchor/quote (e.g., *"Precision is the antidote to liability."*).

## MANDATE
- **Release Only**: This tool is for milestone "releases" only. NEVER run autonomously. Git commits must be explicitly requested by the human user to avoid repository pollution.
- **Non-Interactive**: Always draft the message to a file (e.g., `commit-XXXXXXX.txt`) and use `git commit -a -F <file>`.
- **Clinical & Authoritative**: No filler. High signal.
- **Specific Staging**: Never use `git add <folder>`. Always ask the human to add untracked files, or use `git add -u` for modified tracked files.
- **Verification**: Run `git status` after commit to confirm success.
