# @privateClaw - The Sovereign Privacy Substrate

> *"We don't protect the vault; we delete the target."*

## 🛡️ Overview
**@privateClaw** is a lightweight, high-velocity privacy layer designed to secure AI agent conversations by removing all Personally Identifiable Information (PII) and Intellectual Property (IP) before they leave the local perimeter. It transforms sensitive prompts into logic-preserving "Semantic Mirages" that are safe for cloud-based inference.

## 🚀 Key Features
*   **Zero Material Liability**: No PII/IP is ever stored or transmitted to the cloud.
*   **Deterministic Tokenization**: Stateless masking using hardware-derived keys (HMAC-based). No central vault or database required.
*   **Semantic Error Diffusion**: Dithers identity "energy" into generic archetypes while maintaining the intent and structure of the prompt.
*   **Local Re-hydration**: LLM responses are "refocused" back into cleartext identity material exclusively within the trusted local environment.
*   **Docker-Native**: Runs in a 100MB Docker image on standard CPU hardware (no GPU tax).

## 🛠️ Architecture: The Holographic Inversion
1.  **Dither (Local)**: Identify PII "peaks" (Names, Addresses, Codes) and project them into generic archetypes (e.g., `Michel` -> `The Individual`).
2.  **Mirage (Cloud)**: The LLM receives a valid, high-quality prompt that feels natural but contains zero reversible PII.
3.  **Refocus (Local)**: The agent receives the response and applies an inverse integration filter to "blink" the original secrets back into the UI.

## ⚖️ Liability Removal
By ensuring that sensitive material mathematically does not exist in the cloud, @privateClaw effectively eliminates the liability of possession. In the event of a cloud-side breach, the attacker only gains access to "Meaningless Material"—semantic travesties that cannot be reversed without the user's hardware-bound key.
